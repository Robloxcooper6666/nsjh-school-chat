這裡有你開始發文前需要知道的一切

1. 註冊帳號

發文前請先擁有 GitHub 帳號。它是看板的資料庫來源。

2. 安裝 App

推薦手機安裝 GitHub App，點擊發布後體驗會更順暢。

3. 完成發布

在 GitHub 頁面點擊「Submit new issue」後，訊息才會同步。

常見問題 (FAQ)

Q: 為什麼點擊「發布」後會跳到 GitHub？

為了維持看板的免費運作與資料透明，我們將 GitHub 的 Issues 功能作為「留言資料庫」。跳轉至 GitHub 是為了讓系統確認是你在發文，並將內容正式寫入資料庫中。

Q: 匿名發布真的匿名嗎？

看板介面會顯示為「匿名同學」，但你的 GitHub 帳號名稱仍會出現在 Issue 的原始紀錄中。這能防止惡意霸凌，同時提供一定的隱私感。請遵守校園禮儀，切勿發布攻擊性言論。

Q: 發布後多久會出現在看板上？

看板會每 30 秒自動刷新一次。一旦你在 GitHub 成功點擊「Submit」，通常在一分鐘內就能在首頁看到。

Q: 訊息可以刪除或修改嗎？

如果你需要刪除或修改發言，你需要登入 GitHub 找到該則 Issue 進行編輯。看板會同步顯示修改後的結果。
